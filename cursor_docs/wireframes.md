## Interaction Patterns

### Progressive Disclosure
- **Landing:** Show only essential input field
- **Results:** Reveal details progressively (summary â†’ evidence â†’ actions)
- **Forms:** One section at a time, clear progress indicators

### Information Hierarchy
- **Primary:** Assessment verdict and savings amount (large, prominent)
- **Secondary:** Evidence details and comparable sales (supporting)
- **Tertiary:** Metadata, timestamps, technical details (small, subdued)

### Error Prevention
- **Input validation:** Real-time feedback on address entry
- **Confirmation dialogs:** For destructive actions (delete property)
- **Undo capability:** For reversible actions (remove comparable)

### Loading States
- **Skeleton screens:** Content-shaped placeholders
- **Progress indicators:** For multi-step processes
- **Time estimates:** "This usually takes X seconds"

---

## Accessibility Considerations

### Screen Reader Support
- **Semantic HTML:** Proper heading hierarchy (H1â†’H2â†’H3)
- **ARIA labels:** Form inputs, status messages, buttons
- **Alt text:** Meaningful descriptions for icons and images

### Keyboard Navigation
- **Tab order:** Logical flow through interface elements
- **Focus indicators:** Clear visual focus rings
- **Shortcut keys:** Common actions (Enter to submit, Esc to cancel)

### Touch Targets
- **Minimum size:** 44px for mobile touch targets
- **Spacing:** Adequate space between interactive elements
- **Gestures:** Standard swipe and tap interactions

---

## Implementation Notes

### Component Inventory
Based on these wireframes, we'll need:
- Header/Navigation component
- Card component (various sizes)
- Button component (primary, secondary, ghost)
- Input component (text, autocomplete)
- Form components (layouts, validation)
- Loading/Progress components
- Modal/Dialog components
- Table/List components
- Status/Alert components

### Responsive Breakpoints
- **Mobile:** < 768px (stacked, single column)
- **Tablet:** 768px - 1024px (2-column layouts)
- **Desktop:** > 1024px (multi-column, sidebar layouts)

### Design Tokens
- **Spacing scale:** 4px base unit (4, 8, 12, 16, 24, 32, 48, 64px)
- **Border radius:** 4px (small), 8px (medium), 12px (large)
- **Shadows:** Subtle elevation (1dp, 2dp, 4dp, 8dp)
